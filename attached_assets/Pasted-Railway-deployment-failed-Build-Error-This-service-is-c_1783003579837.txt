## Problem
Service @workspace/api-client-react is a REPO service using the Railpack builder. The build fails at the start command detection stage with "No start command detected." The deployed commit (ab936a92) does not include the railway.json or the start script in package.json that were added in the subsequent commit (c06c33ae). The watchPatterns config (['/lib/api-client-react/**']) caused the fix commit to be SKIPPED.

## Diagnosis
The root package.json at the failing commit is missing the "start" script. The railway.json (which specifies NIXPACKS builder and a startCommand) was not present at this commit. Even on HEAD, the railway.json exists but the package.json start script may be missing. Railpack checks for a start script in package.json, a main field, or an index.js/index.ts — none of these are present for this library package.

## Fix
1. File: `package.json` (repo root)
   Current scripts section:
   ```json
   "scripts": {
     "preinstall": "sh -c 'rm -f package-lock.json yarn.lock; ...",
     "build": "pnpm run typecheck && pnpm -r --if-present run build",
     "typecheck:libs": "tsc --build",
     "typecheck": "pnpm run typecheck:libs && pnpm -r --filter \"./artifacts/**\" --filter \"./scripts\" --if-present run typecheck"
   }
   ```
   Replace with (add start script):
   ```json
   "scripts": {
     "preinstall": "sh -c 'rm -f package-lock.json yarn.lock; ...",
     "build": "pnpm run typecheck && pnpm -r --if-present run build",
     "typecheck:libs": "tsc --build",
     "typecheck": "pnpm run typecheck:libs && pnpm -r --filter \"./artifacts/**\" --filter \"./scripts\" --if-present run typecheck",
     "start": "node artifacts/api-server/dist/index.mjs"
   }
   ```
   This ensures Railpack can detect the start command even without railway.json, and aligns with the start script added in the fix commit (c06c33ae).
